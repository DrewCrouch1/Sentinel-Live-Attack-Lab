# Microsoft Sentinel Live Attack Lab

## Executive Summary

This project demonstrates how to deploy a lightweight **Azure-based attack monitoring lab** using a publicly exposed Windows virtual machine, **Log Analytics Workspace**, and **Microsoft Sentinel** to capture and analyze failed Remote Desktop Protocol (RDP) login attempts.

The objective was to simulate how a cloud security team can collect telemetry from an exposed asset, enrich suspicious authentication activity with geolocation data, and visualize attack trends inside Microsoft Sentinel.

This project highlights practical skills in:

- Azure infrastructure deployment
- Microsoft Sentinel configuration
- Log Analytics ingestion
- PowerShell-based log enrichment
- KQL-based threat hunting and visualization

---

## Lab Objectives

- Deploy an Azure virtual machine to simulate an internet-exposed endpoint
- Configure logging and monitoring through Log Analytics Workspace
- Enable Microsoft Sentinel for security monitoring
- Capture failed RDP authentication attempts
- Enrich attack source IP addresses with geolocation data
- Visualize attacker activity through KQL and Sentinel workbooks/maps

## Project Walkthrough

## 1. Azure Honeypot Deployment

A Windows 10 virtual machine was deployed in Microsoft Azure and intentionally configured as an exposed target to generate authentication telemetry from internet-based scanning and login attempts.

The Network Security Group (NSG) was configured broadly enough to allow inbound traffic to reach the VM, creating a realistic attack surface for monitoring.

## 2. Log Collection and Sentinel Configuration

A **Log Analytics Workspace** was created to collect relevant telemetry from the virtual machine. Microsoft Sentinel was then enabled and connected to the workspace to support centralized monitoring, investigation, and visualization.

This established the pipeline required to search, parse, and analyze security events generated by the VM.

---

## 3. Failed RDP Login Tracking

The lab focused on **Windows Security Event ID 4625**, which records failed logon attempts. Because the system was internet exposed, failed authentication attempts began accumulating from external IP addresses.

This provided a useful dataset for basic threat hunting and attack trend analysis.

## 4. Geolocation Enrichment with PowerShell

A PowerShell script was used to extract failed logon events and enrich source IP address data through the **ipgeolocation.io** API.

The enrichment added useful context such as:

- Country
- State / Province
- City
- Latitude / Longitude
- ISP / Company data

This step transformed raw authentication failures into more useful analyst-readable telemetry for investigation and mapping.
